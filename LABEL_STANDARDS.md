# LABEL STANDARDS — ระบบรถรับส่งนักเรียนจังหวัดลำปาง

เอกสารนี้เป็นมาตรฐานการตั้งชื่อสำหรับ **UI labels** ทั้งระบบ
ใช้อ้างอิงเมื่อสร้างหรือแก้ไข: หัวข้อหน้า, card title, chart title, section title, table header, status badge, empty state message และข้อความสั้นที่ผู้ใช้มองเห็นบนหน้าจอ

> **ขอบเขต:** เอกสารนี้ใช้กับ hardcoded UI text เท่านั้น ไม่ใช่การเปลี่ยนชื่อจริงที่มาจากฐานข้อมูล

---

## 1. หลักการตั้งชื่อ

1. **ความหมายเดียวกัน ใช้คำเดียวกัน** — ถ้า "ส่งเช้า" หมายถึง morning check-in ทุก role ต้องใช้คำนี้เหมือนกัน
2. **ใช้ภาษาที่ผู้ใช้เข้าใจทันที** — เลือกคำที่ครู, คนขับ, และผู้บริหารอ่านแล้วรู้ความหมายโดยไม่ต้องถาม
3. **หลีกเลี่ยงคำกำกวม** — เช่น "ยังไม่ครบ" ไม่ระบุว่าไม่ครบเรื่องอะไร ให้ใช้ "ยังมีรายการค้าง" แทน
4. **ข้อมูลจากฐานข้อมูล ห้าม rename** — ถ้าชื่อหน่วยงานจริงคือ "สำนักงานเขตพื้นที่การศึกษาประถมศึกษาลำปาง เขต 1" ให้แสดงตามจริง
5. **คำในระดับ action กับ summary อาจต่างกันได้** — เช่น ปุ่มอาจใช้ "ส่งถึงโรงเรียน" แต่ summary ใช้ "ส่งเช้า" ทั้งคู่ถูกต้องตราบที่คงเส้นคงวาในบริบทของตัวเอง

## 5. คำที่ควรหลีกเลี่ยง

| คำเดิม | ใช้แทน | เหตุผล |
|--------|--------|--------|
| เขตพื้นที่ (เป็น label) | สังกัด | สั้นกว่า ชัดกว่า ครอบคลุมกว่า |
| ลาวันนี้ | นักเรียนลา | "วันนี้" ซ้ำซ้อนกับ subtitle |
| นักเรียนลาวันนี้ | นักเรียนลา | เหตุผลเดียวกัน |
| ดำเนินการแล้ว | สำเร็จแล้ว | สั้นกว่า อ่านง่ายกว่า |
| ยังไม่ครบ | ยังมีรายการค้าง | ระบุชัดว่าค้างเรื่องอะไร |
| สถานะรายคัน | สถานะรถแต่ละคัน | เป็นทางการกว่า อ่านชัดกว่า |
| รถที่รอดำเนินการมากที่สุด | รถที่มีรายการค้างมากที่สุด | สอดคล้องกับคำว่า "ค้าง" |
| ภาพรวมจังหวัดลำปาง | ภาพรวมจังหวัด | ไม่ hardcode ชื่อจังหวัด |
| ภาพรวมเขตพื้นที่ | ภาพรวมสังกัด | คงเส้นคงวากับ "สังกัด" |

## 6. ข้อยกเว้นสำคัญ

### ข้อมูลจริงจากฐานข้อมูล
ชื่อหน่วยงาน เช่น `สำนักงานเขตพื้นที่การศึกษาประถมศึกษาลำปาง เขต 1` คือข้อมูลจริงจาก DB
ห้ามเปลี่ยนเป็น "สังกัดลำปาง เขต 1" — ให้แสดงตามที่เก็บในฐานข้อมูล
ถ้าต้องย่อเพื่อแสดงในกราฟ ให้ใช้ `string.replace()` ตัดคำนำหน้าออก ไม่ใช่เปลี่ยนข้อมูลต้นทาง

### คำเฉพาะสิทธิ์ driver
สิทธิ์ driver ใช้ภาษา operational ที่แตกต่างได้ เช่น:
- "ภาพรวมวันนี้" แทน "ภาพรวมคนขับ"
- "ลาวันนี้" ในหัว section (ยอมรับได้ เพราะ driver ดูแค่วันนี้)
- "ส่งถึงโรงเรียน" / "รับกลับบ้าน" สำหรับปุ่ม action

แต่ status badge ต้องคงเส้นคงวา: "ลา", "รอ", "สำเร็จแล้ว" ตามมาตรฐานกลาง

## 7. ตัวอย่างการใช้งาน

### Card label
```
label="นักเรียนทั้งหมด"    ← ถูก
label="จำนวนนักเรียน"       ← หลีกเลี่ยง (ไม่ใช่คำมาตรฐาน)
label="นักเรียนลา"          ← ถูก
label="ลาวันนี้"             ← หลีกเลี่ยง
```

### Chart title
```
<p>สถานะส่งเช้า</p>          ← ถูก
<p>สถานะส่งเช้าจังหวัด</p>    ← หลีกเลี่ยง (ซ้ำซ้อนกับหน้า)
```

### Table title
```
<h2>สถานะรถแต่ละคัน</h2>     ← ถูก
<h2>สถานะรายคัน</h2>          ← หลีกเลี่ยง
```

### Empty state
```
<p>ไม่พบรถที่ค้นหา</p>        ← ถูก
<p>ไม่มีข้อมูล</p>             ← ใช้ได้กับ generic case
<p>ไม่มีข้อมูลแนวโน้ม</p>      ← ใช้กับ trend chart
```

## 8. แนวทางก่อน merge งาน UI

ก่อน merge PR ที่แตะหน้า dashboard หรือ UI ที่ผู้ใช้เห็น ให้ตรวจ checklist นี้:

- [ ] ไม่มีคำเก่าที่อยู่ในรายการ "คำที่ควรหลีกเลี่ยง" หลงเหลือ
- [ ] คำใน page title, card, chart, table ใช้ภาษาเดียวกันกับมาตรฐานในเอกสารนี้
- [ ] subtitle / helper text ช่วยให้เข้าใจตัวเลขชัดขึ้น ไม่ซ้ำซ้อนกับ label
- [ ] ไม่มีการเปลี่ยนชื่อจริงจากฐานข้อมูลโดยไม่ตั้งใจ
- [ ] status badge ใช้คำเดียวกันทุก role: "รอ", "ลา", "ครบ"
- [ ] empty state message ชัดเจนว่า "ไม่มีข้อมูล" หรือ "ค้นหาไม่พบ"
- [ ] ข้อความใหม่ไม่ทำให้ layout ล้นจอบน mobile (375px)
